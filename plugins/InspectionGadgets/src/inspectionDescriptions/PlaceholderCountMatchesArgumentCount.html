Reports SLF4J or Log4j 2 logging calls, such as logger.info("{}: {}", key) where the number of {} placeholders in the logger message doesn't match the number of other arguments to the logging call.

Use the inspection option, if you use SLFJ4 with logging frameworks, which allow to have a placeholder for the last parameter with an exception type. For example: 


log.error("For id {}: {}", id, new RuntimeException()); // will not be reported
This option works only for SLF4J.
